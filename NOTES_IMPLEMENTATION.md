# Implementation notes

Короткие заметки по текущей реализации `gpp_frc_repo`.

## Производственный профиль

Публичный конфиг намеренно минимальный. Пользователь задаёт только `target_packages`, а вся логика генерации и параметры качества остаются внутри кода. Это нужно, чтобы модуль работал как единая системная генерация без ручного выбора режима.

## Backend auto

Auto mode устанавливает Vulkan и GLES hooks одновременно. Активным становится тот backend, который первым подтверждает реальный present:

- Vulkan: `vkQueuePresentKHR`;
- GLES: `eglSwapBuffers`.

После выбора backend не переключается в рамках текущего процесса.

## Vulkan generation

Vulkan путь использует compute shader и промежуточные GPU images:

- `gPrev` — предыдущий кадр;
- `gCur` — текущий кадр;
- `gOut` — сгенерированный midpoint.

CPU readback не используется. Это снижает задержку и избегает лишней нагрузки на память.

## GLES generation

GLES путь использует fullscreen shader и две history textures. Перед любыми операциями сохраняется GL state приложения, после генерации он восстанавливается. Это важно для игр, которые оставляют нестандартные blend/depth/scissor/FBO/VAO состояния перед `eglSwapBuffers`.

Захват текущего кадра предпочитает `glBlitFramebuffer`, потому что на некоторых драйверах direct copy из default framebuffer может быть недоступен или возвращать `GL_INVALID_OPERATION`.

## Pacing

В текущей стабильной версии не используются presentation timestamp hints для GLES. Реальный кадр приложения остаётся главным источником pacing, а generated frame вставляется перед ним. Это оказалось стабильнее для динамичных сцен.

## Качество изображения

Adaptive blend сохраняет небольшое размытие в динамике, потому что оно визуально скрывает разрывы между кадрами и делает быстрые движения мягче. Для мелких элементов используется защита деталей и ограничение цвета по соседним пикселям, чтобы уменьшить пятна на насыщенных сценах.
